    🔁 핵심 업무 흐름 (필드 수준 상세)

    A. 신제품 등록 — 여기가 정확해야 모든 계산이 정확해집니다
    1
    원료 등록 — 이름, 단위(g), 표준단가(원/g 기준!), 유통기한 일수 입력. kg당 가격을 알면 ÷1000. 향료만이 아니라 에탄올·정제수까지 전부 등록해야 배합비 100%를 채울 수 있습니다
    2
    포장재 등록 — 병·캡·박스·라벨 각각 개당 표준단가와 함께
    3
    제품 등록 — 판매가 입력, 충진중량은 실제 내용물 무게(g): 변환 도우미에 용량(ml)×비중(향수 약 0.87) 입력 후 [→ g 적용]. BOM 구성에서 원료는 배합비(%), 포장재는 개수로 추가 — 배지가 "원료 합계 100% ✓" 초록색이 되게 맞추면 표준원가가 자동 계산됩니다
    4
    포뮬러 버전 저장 — 확정 배합을 v1으로 저장하면 그 시점 처방이 스냅샷으로 영구 보존. 이후 배합을 바꾸면 v2로 새로 저장 (이력 관리). [상세] 버튼에서 성분별 비고·숙성조건 메모까지 기록
    B. 구매 → 입고 → 검사
    1
    발주 등록 — 공급처 선택 → 품목·수량·단위(g/kg/ea)·단가 입력. kg 선택 시 자동으로 g 환산 저장되어 BOM 계산과 단위가 맞습니다. 도착하면 [입고완료] 클릭 → 입고기록과 LOT이 자동 생성
    2
    직접 입고 — 발주 없이 들어온 자재는 여기서. 품목·수량(단위 선택 가능)·실구매 단가·유통기한 입력. 이 단가가 나중에 벌크·완제품 실제원가의 재료가 되므로 정확하게
    3
    수입검사 — 새 LOT은 HOLD 상태. 검사 후 합격 처리해야 생산에 사용 가능
    C. 생산 사이클 — 원가가 자동으로 따라옵니다
    1
    MRP 계산 — 제품 선택 (수량 기본 100, 수정 가능) → 원료·포장재 필요량과 현재고·부족분이 즉시 표시. 부족분은 발주로
    2
    벌크배합 — 제품·수량(생산 예정 개수)·숙성일 입력 → 원료가 유통기한 임박순(FEFO)으로 자동 차감되고, 소모된 원료 LOT들의 실구매가 합산으로 벌크 원가/ea 자동 기록
    3
    숙성 확인 — 벌크숙성추적 또는 벌크 목록의 숙성완료일 확인. 완료된 벌크만 충진 진행
    4
    충진/생산배치 — 벌크 LOT 선택·충진수량 입력 → 포장재 자동 차감, 완제품 실제원가/ea = 벌크원가 + 포장재 실구매가로 자동 기록. 완제품 LOT은 HOLD로 생성
    5
    생산품검사 — 외관·향·용량 검사 후 합격 → OK 상태가 되어 출고 가능
    D. 판매 → 견적 → 월말 마감
    1
    견적서 — 고객사·품목·수량 선택 (단가는 제품 판매가 자동) → 인쇄/PDF 또는 [메일로 보내기]로 견적 본문이 담긴 메일 초안 발송
    2
    출고/판매 — 제품·수량 입력 → QC 합격(OK) LOT만 FEFO 순서로 자동 차감. HOLD LOT은 출고 불가
    3
    수불부 대조 — 유형(원료/포장재/완제품)별 입고·소모·출고·조정 전체 이력과 누적 잔량. CSV로 내보내 회계 자료로 보관
    4
    월말 실사 — 실물 재고와 시스템 잔량을 대조하고 차이는 재고조정으로 기록 (사유 필수). 연간 실적은 연도·월별 보고서에서 확인

    제품 마스터 + BOM(배합비/포장재) + 충진중량 + 로스율 → 표준원가·마진율 자동 계산. [N개·수정] 버튼으로 BOM 팝업 편집

    제품표준서

    표준원가/마진율이 0으로 나와요
    원료·포장재 마스터에 표준단가가 비어 있으면 0으로 계산됩니다. 원료는 원/g 기준으로 채워주세요 (kg당 가격 ÷ 1000).
    출고가 안 돼요 (재고는 있는데)
    해당 LOT이 HOLD(검사 대기) 상태입니다. 5·품질 메뉴에서 합격 처리하면 출고할 수 있습니다.
    재고 단가와 표준원가가 달라요
    정상입니다. 표준원가는 설계 목표(현재 BOM×표준단가), 재고 단가는 실제 결과(생산 당시 실구매가). 차이가 크면 구매가 인상 또는 배합 변경 신호입니다.
    원료를 kg으로 샀는데 어떻게 입력하나요
    발주·입고 화면에서 단위를 kg으로 선택하고 kg 수량·kg당 단가를 그대로 입력하면 시스템이 g 기준으로 자동 환산해 저장합니다.
    아침 메일 리포트가 안 와요
    GitHub 저장소 → Actions 탭에서 SHIFTI Daily Report 실행 기록을 확인하세요. 빨간 X면 Secrets(서비스 키·앱 비밀번호) 설정을 점검하고, 노즈에게 화면을 보여주시면 바로 진단해 드립니다.
